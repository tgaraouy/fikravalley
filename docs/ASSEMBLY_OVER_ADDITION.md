**The Gap**: We're adding more commitments to already stretched individuals.

## Reframing: Assembly Over Addition

### ‚ùå Current "Addition" Mindset (Risky)

```
New Features = 
  + Moroccan priority tags
  + Mentor matching system  
  + PDF adoption kits
  + WhatsApp group automation
  + 3-week validation sprint
  + SDG alignment (secondary)
```

**Problem**: Each feature adds cognitive load, requires learning, creates commitment.

### ‚úÖ "Assembly" Mindset (What Works)

```
Assembly = 
  WhatsApp (already used 4h/day) 
  + AI agent that drafts messages to customers
  + Mentor drops in *only when needed*
  + 3-DH validation via existing mobile money
  + Moroccan priorities as *filter*, not homework
```

**Solution**: Work with existing behaviors, remove friction, create headspace.

## Key Principles

### 1. Focus on Assembly, Not Addition

**Before**: "Adopt this idea and we'll give you a mentor, PDF, group chat, and validation roadmap"

**After**: "You already chat with customers on WhatsApp. Here's an AI that drafts validation messages in Darija. A mentor from Montreal is auto-added only when you get stuck."

### 2. Create Headspace (The Most Valuable Question)

**Golden Question**: "What can I do to give you headspace?"

**For Adopters**:
- Auto-detect budget/skill level (no forms)
- Pre-draft mentor outreach messages
- Handle bureaucratic parts of validation
- Use WhatsApp as primary interface

**For Mentors**:
- ONE weekly digest email (not dashboard)
- "Sofia in Kenitra needs 15 min. Reply 'yes' to help."
- No browsing 500 ideas

### 3. Treat Partnerships Like Governance

**Structure**:
- Adopter + Mentor + Fikra Valley Community Manager (3 people max)
- 15 minutes, WhatsApp voice notes (not Zoom)
- One question: "What blocked you this month, and what assembly do we need?"

### 4. Listen and Adapt (The Invisible Shields)

**Recognize Survival Mechanisms**:
- "It's not my mandate" ‚Üí Position as informal side project
- "Procurement takes months" ‚Üí Use mobile money only
- "That's not how we do it" ‚Üí Frame as "digital twist on traditional" not "disruptive innovation"
